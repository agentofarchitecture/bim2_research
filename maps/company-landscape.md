Last updated: 2026-05-24.

Purpose: maintain a grounded company/product map for BIM 2.0 so the people map is not floating in abstract research. The strongest source anchors are ArchiTech Network / ATN, especially Oliver / Olly Thomas's BIM 2.0 company graph, plus AEC Magazine's independent BIM 2.0 / Rebuilding BIM coverage and primary company sources.

## Short answer: top BIM 2.0 company set

### Core BIM 2.0 / Rebuilding BIM authoring competitors

These are the companies most directly trying to rebuild BIM authoring/collaboration itself:

1. **Snaptrude** — browser/cloud BIM and AI-assisted concept-to-BIM workflow.
2. **Arcol** — collaborative web-native architectural design / “Figma for AEC” wedge.
3. **Motif** — intelligent browser workspace and BIM-reinvention thesis from former Autodesk/Revit/AutoCAD leadership; visible product currently collaboration/review/AI workspace more than full authoring.
4. **Qonic** — cloud BIM modelling, construction/detail BIM and component-level model data.

### BIM 2.0-adjacent automation / AI-to-BIM / schematic-to-BIM

5. **Hypar** — computational design automation, functions/API model, Revit-compatible building design automation.
6. **Skema** — BIM workflow compression: schematic design to coordinated BIM/Revit using firm/project knowledge.
7. **Finch / Finch3D** — AI/graph-based early design optimisation and floor-plan feedback.
8. **Augmenta** — autonomous building-systems / MEP design automation, especially electrical.
9. **SWAPP** — automated BIM/construction-documentation production; strong AEC Magazine signal but primary site blocked from this environment.
10. **Higharc** — cloud-native homebuilding platform; 2025 coverage highlights AI converting 2D sketches into 3D BIM models.
11. **Illoca** — AI-native architectural design: sketch/image/text to BIM-compatible 3D massing and agentic layout editing; early but highly relevant.
12. **Codesign** — concept-stage design with BIM integration, carbon and generative AI features.
13. **qbiq** — AI commercial real-estate space planning that claims floor plans, tours and BIM models in hours.

### Upstream feasibility / planning / developer workflows

14. **Giraffe** — AI-driven real-estate orchestration layer across feasibility, planning, design and finance.
15. **TestFit** — feasibility/site-planning/pro-forma automation.
16. **Spacio.ai** — ATN-listed AI-powered building design software; primary-source verification still needs deeper pass.
17. **Archistar** — AI planning/feasibility platform for property development.
18. **cove / cove.tool** — performance-driven early design and development intelligence.
19. **canibuild** — site suitability/feasibility automation for builders.
20. **Laiout** — automated floor-planning software.
21. **ArkDesign.ai** — generative schematic design / code-aware planning; needs stronger primary validation.

### Data / interoperability / open web BIM infrastructure

22. **Speckle** — open AEC data/connectors/project data layer; crucial BIM 2.0 infrastructure rather than direct authoring replacement.
23. **That Open Company / That Open Engine** — open web-BIM/IFC developer components and apps.
24. **IfcOpenShell / Bonsai** — open-source IFC/BIM implementation infrastructure; not a startup comparator but strategically essential.

### Adjacent authoring-tool AI, documentation knowledge and product-data workflows

25. **Rayon** — collaborative browser CAD/planning/documentation; more CAD than BIM unless semantics deepen.
26. **RunChat** — Rhino/GH agentic scripting and workflow canvas; direct Rhino Mind comparator, but not a BIM platform by itself.
27. **NonicaTab** — Revit + AI connector pattern; verify in a dedicated pass.
28. **OLI** — product/manufacturer BIM object data workflow; verify in a dedicated pass.
29. **EvolveLAB** — AEC AI tools such as Veras/Glyph/Morphis; acquired by Chaos, strong ecosystem signal but more tool/plugin layer than BIM 2.0 platform.
30. **Pirros** — architectural/detail knowledge and documentation intelligence; not BIM authoring, but very relevant to firm memory and repeatable details.
31. **UpCodes** — AI/code compliance layer; adjacent now, but important if BIM 2.0 becomes model-to-compliance automation.
32. **Gendo** — architecture AI visualisation; watchlist only for BIM 2.0.
33. **Infurnia** — cloud BIM modeller; older but useful non-incumbent comparison.
34. **Autodesk Forma** — incumbent early-stage cloud planning/design platform; strategic pressure point rather than startup.

## Strategic read for Rhino Mind

The company map confirms BIM 2.0 is not one category. It is a bundle of competing wedges:

- collaborative cloud authoring;
- test-fit / feasibility intelligence;
- schematic-to-BIM automation;
- open data/interoperability;
- AI-assisted design exploration;
- automated documentation / detail knowledge;
- product/manufacturer data integration;
- agentic tool control inside existing authoring tools.

Rhino Mind’s likely wedge is different: **contextual tooling and project memory inside Rhino/Grasshopper practice**, with enough semantic structure to connect to Speckle/IFC/IDS later. The biggest warning is not to compete as “another BIM authoring platform”. The opportunity is to become the **semantic sidecar** to existing Rhino/Grasshopper workflows: understand the project dialect, retrieve office tools and precedents, run checks, and produce evidence.
