# BIM 2.0 Research Loop

A living research repository maintained by Steve for Luke / Agents of Architecture.

## Purpose

Develop a fine-grained, meta-level understanding of **BIM 2.0**: where digital building practice is going, what is technically possible now, where AI/agents/spatial computing/data standards are pushing the field, and what Agents of Architecture / Rhino Mind can do about it.

## Core question

> What comes after BIM as file/model coordination — and how can architects use agents, semantic project intelligence, and tooling-on-demand to create a practical advantage?

## Daily loop

A scheduled Steve job runs daily and updates this repo with:

- New signals from research, standards, startups, vendors, and AEC practice
- A short daily intelligence note
- Updates to the evolving map of BIM 2.0 themes
- Opportunities and experiments for Agents of Architecture / Rhino Mind
- Open questions to investigate next

## Repository structure

- `daily/` — daily research notes, one markdown file per run
- `maps/` — evolving synthesis maps and taxonomies, including `maps/company-landscape.md`
- `sources/` — tracked sources, search queries, feeds, watchlists, and `sources/youtube-atn-grounding.md`
- `opportunities/` — practical things we could build, test, or commercialise
- `briefs/` — higher-level periodic summaries

## Current working definition

**BIM 2.0** is the shift from static coordinated building models toward live, semantic, agent-readable project systems: models, drawings, specs, constraints, site data, regulations, costs, fabrication logic, and operational feedback become queryable and actionable by humans and AI agents.

Likely ingredients:

- Semantic building data and knowledge graphs
- Open standards: IFC, IDS, bSDD, Speckle, USD/glTF where relevant
- AI copilots and autonomous agents for modelling/documentation/QA
- Retrieval over project memory, precedent, code, specs, and reusable details
- Tooling-on-demand inside authoring environments like Rhino/Revit
- Automated checking: compliance, coordination, carbon, cost, buildability
- Human-in-the-loop design intelligence rather than fully automated architecture

## Steve’s research stance

Track both:

1. **Fine grain:** concrete papers, tools, standards, companies, implementation patterns.
2. **Meta grain:** structural shifts, adoption barriers, business models, and strategic openings for Rhino Mind / Agents of Architecture.
